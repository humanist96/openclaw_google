Step 01

환경 준비

AntiGravity(VS Code 기반 AI 에디터)를 설정하고 작업 환경을 준비합니다.

ℹ️
사전 준비
AntiGravity가 설치되어 있어야 합니다. 공식 사이트에서 설치 방법을 확인해주세요.

Windows 사용자: WSL 연결

Windows 사용자는 반드시 WSL(Windows Subsystem for Linux)로 연결해야 합니다. Mac 사용자는 이 단계를 건너뛰세요.

명령 팔레트 열기 Ctrl + Shift + P를 눌러 메뉴 검색창을 엽니다.
WSL 검색 및 연결 검색창에 WSL을 입력하고, Remote WSL: Connect WSL using Distro를 선택합니다.
Ubuntu 배포판 선택 설치된 Ubuntu 배포판을 선택하면 WSL 환경에서 실행됩니다.
📷
WSL 연결 화면 스크린샷
해당 스크린샷 이미지를 준비하여 교체하세요

작업 폴더 생성

사용자 홈 디렉토리에서 작업 폴더를 만듭니다.

bash

File > Open Folder를 선택하고, 방금 만든 ~/workspace/openclaw 폴더를 열어주세요.

터미널 열기

터미널 패널 열기 Ctrl + J 또는 상단 메뉴에서 Toggle Panel을 클릭합니다.
터미널 전체화면으로 확대 Move Terminal into Editor Area를 클릭하면 큰 화면에서 편리하게 작업할 수 있습니다.

Step 02

OpenClaw 설치 스킬 다운로드

AntiGravity에서 사용할 GCP OpenClaw 스킬을 설치합니다.

GCP OpenClaw 플러그인을 설치합니다. 아래 명령어를 실행하면 설치 스킬이 자동으로 과정을 안내합니다.

스킬 설치 명령어

bash 
npx @openclaw/gcp-setup install

설치 확인 메시지가 나타나면 Y를 입력합니다.

📷
스킬 설치 완료 화면

불필요한 파일 정리

설치가 완료되면 .agents 폴더가 생성됩니다. GCP OpenClaw 스킬만 남기고 나머지는 삭제해주세요.

💡
스킬 확인
AntiGravity 우측 패널에서 "사용 가능한 스킬을 확인해 줘"라고 입력하면 GCP OpenClaw 스킬이 검색됩니다. 이 스킬을 통해 또는 아래 매뉴얼을 따라 직접 설치를 진행하실 수 있습니다.
✅
체크포인트

Step 03

구글 클라우드 설정

프로젝트 생성, API 활성화, OAuth 인증 정보를 설정합니다.

3-1. 새 프로젝트 생성

Google Cloud Console 접속 console.cloud.google.com에 접속하여 로그인합니다.
새 프로젝트 만들기 상단의 프로젝트 선택 버튼을 클릭하고, 새 프로젝트 버튼을 누릅니다. 프로젝트 이름을 openclaw로 입력합니다.
프로젝트 선택 생성 완료 후 알림이 뜨면, 프로젝트 선택 버튼을 눌러 방금 만든 프로젝트를 선택합니다.
📷
구글 클라우드 프로젝트 생성 화면
해당 스크린샷 이미지를 준비하여 교체하세요

3-2. API 활성화 (5개 서비스)

API 및 서비스 메뉴로 이동하여 아래 5개 API를 각각 검색하고 사용 설정 버튼을 눌러주세요.

서비스API 이름용도
GmailGmail API메일 읽기/쓰기/삭제
캘린더Google Calendar API일정 관리
시트Google Sheets API스프레드시트 제어
독스Google Docs API문서 작성
드라이브Google Drive API파일 관리
📷
API 활성화 화면
해당 스크린샷 이미지를 준비하여 교체하세요

3-3. OAuth 동의 화면 구성

동의 구성 메뉴 진입 API 및 서비스 > OAuth 동의 화면 > 동의 구성을 클릭하고 시작하기 버튼을 누릅니다.
앱 정보 입력 앱 이름: openclaw, 사용자 지원 이메일: 본인 이메일 선택, 사용자 유형: 외부 선택, 연락처 이메일: 본인 이메일 입력 후 만들기 클릭.
테스트 사용자 추가 대상 탭에서 테스트 사용자에 본인 이메일 주소를 추가하고 저장합니다.

3-4. 데이터 액세스 (범위 설정)

범위 추가 클릭 데이터 액세스 탭에서 범위 추가를 클릭합니다.
API별 권한 추가 API 필터에서 Gmail API를 선택하고 필요한 권한을 체크합니다. 같은 방식으로 Calendar, Sheets, Docs, Drive API의 권한도 추가합니다.
업데이트 및 저장 모든 권한을 추가한 후 업데이트 버튼을 누르고 저장합니다.
💡
권한 범위 설정
각 API별로 필요한 권한만 선택하는 것이 좋습니다. 전체 권한을 부여할 수도 있지만, 보안을 위해 필요한 범위만 설정하세요.

3-5. 클라이언트 ID 생성

클라이언트 메뉴로 이동 API 및 서비스 > 사용자 인증 정보 > 클라이언트에서 OAuth 2.0 클라이언트 ID를 만듭니다.
데스크톱 앱 선택 애플리케이션 유형을 데스크톱 앱으로 선택하고, 이름을 openclaw으로 입력합니다.
JSON 파일 다운로드 만들기를 누르면 클라이언트 ID와 시크릿이 생성됩니다. JSON 다운로드 버튼을 눌러 파일을 저장합니다.
🚨
보안 주의
클라이언트 ID와 시크릿은 절대 외부에 노출하지 마세요. 타인이 알고 있으면 안 됩니다.

3-6. 키 파일 저장

프로젝트 폴더에 .keys 폴더를 만들고, 다운로드한 JSON 파일을 복사합니다.

bash 
mkdir -p .keys
# 다운로드한 JSON 파일을 .keys 폴더로 복사하세요
cp ~/Downloads/client_secret_*.json .keys/

3-7. 프로젝트 ID 확인

구글 클라우드 콘솔 상단의 프로젝트 선택 버튼을 클릭하면 프로젝트 ID를 확인할 수 있습니다. 이 ID를 메모해두세요.

Step 06

gog-cli와 OpenClaw 설치

VM 서버에 SSH로 접속하여 OpenClaw과 gog-cli를 설치합니다.

VM에 SSH 접속

gcloud 명령어로 VM에 접속합니다. 인스턴스 이름과 존(zone)은 실제 값으로 교체하세요.

bash 
gcloud compute ssh [인스턴스명] --zone=[존] --project=[프로젝트ID]

OpenClaw 설치

VM 내부에서 OpenClaw를 설치합니다. AntiGravity 스킬이 자동으로 수행하지만, 수동으로 진행할 경우 아래 명령어를 사용하세요.

ℹ️
스킬 자동 설치
AntiGravity 스킬이 이 과정을 자동으로 수행합니다. 만약 자동 설치가 실패하면 아래 매뉴얼 방법으로 진행하세요.

gog-cli 설치

gog-cli는 Linux에서 CLI를 통해 구글의 모든 서비스를 API로 사용할 수 있는 도구입니다. OpenClaw가 구글 서비스를 사용하는 핵심 연결 고리입니다.

💡
gog-cli란?
OpenClaw는 MCP가 아닌 CLI 스킬을 통해 구글 서비스를 사용합니다. gog-cli가 설치되면 Gmail, Calendar, Sheets, Docs, Drive 등 모든 구글 서비스를 명령줄에서 제어할 수 있습니다.

인증 키 복사

앞서 다운로드한 OAuth 인증 JSON 파일을 VM으로 복사합니다.

bash 
# 로컬에서 VM으로 파일 복사 (SCP 사용)
gcloud compute scp .keys/client_secret_*.json [인스턴스명]:~/.keys/ \
  --zone=[존] --project=[프로젝트ID]

gog-cli 인증

VM에 접속한 상태에서 인증을 진행합니다.

bash 
gogh auth add --manual

인증 링크가 나타나면 브라우저에서 열고, 모든 서비스를 선택한 뒤 계속을 눌러주세요. 나타나는 리다이렉트 주소를 복사하여 터미널에 붙여넣으면 인증이 완료됩니다.

📷
gog-cli 인증 완료 화면
해당 스크린샷 이미지를 준비하여 교체하세요
✅
체크포인트
VM에 OpenClaw과 gog-cli가 설치되고, 구글 인증이 완료된 상태이면 성공입니다.
Step 07

OpenClaw 및 텔레그램 봇 설정

OpenClaw 온보딩을 진행하고 텔레그램 메신저 봇을 연결합니다.

7-1. OpenClaw 온보딩

VM에 SSH 접속한 상태에서 온보딩 마법사를 실행합니다.

bash 
openclaw onboard

온보딩 설정 순서

Continue > Yes 시작 화면에서 계속 진행합니다.
QuickStart 선택 설치 방식을 선택합니다.
LLM 모델 선택: AntiGravity Auth 구글 구독 계정을 사용하려면 AntiGravity OAuth를 선택합니다. (OpenAI, Anthropic 등 다른 API도 선택 가능)
모델 선택: Gemini 3 Flash 속도가 빠르고 적당한 성능의 Gemini 3 Flash를 선택합니다.
💡
다른 모델 사용하기
OpenAI API가 있으면 OpenAI를, Claude를 쓰려면 Anthropic을 선택하세요. 지난 영상에서 소개한 Z.AI의 GLM 4.7도 사용 가능합니다.

7-2. 텔레그램 봇 생성

온보딩에서 메신저를 Telegram으로 선택한 뒤, 봇을 생성합니다.

BotFather 검색 텔레그램(웹/앱)에서 검색창에 BotFather를 검색하여 선택합니다.
새 봇 만들기 /newbot 명령어를 입력하거나 클릭합니다.
봇 이름 설정 고유한 이름을 설정합니다. (예: MyOpenClawBot) 반드시 Bot 또는 bot으로 끝나야 합니다.
토큰 복사 봇이 생성되면 표시되는 토큰 링크(세 번째 링크)를 클릭하여 복사합니다.
📷
BotFather 봇 생성 화면
해당 스크린샷 이미지를 준비하여 교체하세요

7-3. 온보딩 마무리

토큰 입력 복사한 텔레그램 봇 토큰을 온보딩 화면에 붙여넣습니다.
스킬 설치 스킬 설정 여부: Yes, Homebrew Install: No, 패키지 매니저: npm, 빌트인 스킬에서 ClawHub를 선택하여 설치합니다.
추가 API 키 설정 Google Places, Gemini, OpenAI, 11Labs 등은 모두 No로 건너뜁니다. (필요시 나중에 추가 가능)

7-4. 퍼소나 설정

OpenClaw이 처음 실행되면 퍼소나(성격)를 설정하라고 합니다. 봇의 이름, 말투, 이모지 사용 여부 등을 자유롭게 설정하세요.

퍼소나 설정 예시 
이름: 단단이
성격: 친절하고 유쾌한 AI 비서
말투: 존댓말 사용
이모지: 자주 사용

7-5. 텔레그램 봇 시작 및 사용자 승인

봇 채팅 시작 BotFather가 보내준 첫 번째 링크를 클릭하고 Start 버튼을 누릅니다.
사용자 승인 처음 메시지를 보내면 승인이 필요합니다. TUI에서 해당 유저를 승인해주세요.
⚠️
보안 기능
승인되지 않은 사용자는 봇과 대화할 수 없습니다. 소유자가 직접 승인해야만 통신이 가능합니다.
✅
체크포인트
텔레그램에서 봇에게 메시지를 보내고 답변이 오면 모든 설정이 완료된 것입니다!
